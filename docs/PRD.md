# 產品需求文件 (PRD)：任務管理系統

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，有許多瑣碎的事項需要處理與追蹤。為避免遺漏重要事項，需要一個簡單直覺的工具來記錄與管理日常代辦任務。
- **目標用戶**：需要進行日常任務與時間管理的使用者（如學生、上班族等）。
- **核心價值主張**：提供一個輕量、反應快速且易於上手的代辦事項管理工具，幫助使用者有效追蹤進度，提升個人生產力並減輕心理負擔。

## 2. 功能需求
系統主要功能與對應的使用者故事（User Stories）如下：

- **功能一：新增任務**
  - 作為使用者，我希望「能夠輸入並新增任務內容」，以便「隨手記錄下需要完成的事項」。
- **功能二：標記完成任務**
  - 作為使用者，我希望「能夠勾選或點擊按鈕將特定任務標記為已完成」，以便「清楚追蹤哪些事項已經處理完畢」。
- **功能三：顯示任務清單**
  - 作為使用者，我希望「能夠一覽無遺地查看目前所有任務的清單（包含已完成與未完成）」，以便「快速掌握我目前的任務狀態」。
- **功能四：刪除任務**
  - 作為使用者，我希望「能夠刪除不再需要執行或是誤輸入的任務」，以便「保持任務清單的乾淨整齊，免受不需要的資訊干擾」。
- **功能五：取消完成標記功能**
  - 作為使用者，我希望「能夠將不小心標記為已完成的任務恢復為未完成」，以便「對於還需處理的任務繼續執行並持續追蹤」。

## 3. 非功能需求
- **技術限制**：
  - 使用 **Flask** 作為後端輕量級框架。
  - 畫面採 **Jinja2** 模板進行渲染。
  - 使用 **SQLite** 作為資料庫，以維持系統的輕量性且便於部署。
- **效能考量**：操作介面應該直覺且反應迅速，網頁載入時間與各種操作之回應時間（如新增、刪除、狀態切換）應盡可能降低，提供流暢的體驗。
- **安全考量**：基本的前後端輸入驗證，防止惡意語法輸入造成 SQL Injection (SQL 注入) 與 XSS (跨站腳本攻擊)。

## 4. MVP 範圍

### Must Have (必須擁有，核心功能)
- 網頁介面的任務清單顯示。
- 表單輸入以新增任務。
- 任務狀態的轉換（待辦 <-> 已完成）。
- 永久移除單一任務的按鈕。

### Should Have (應該擁有，提升體驗)
- 介面上能以視覺方式明顯區分「已完成」與「未完成」事項（例：已完成項目加上刪除線背景反灰）。
- 基本的響應式網頁設計（RWD），至少確保在手機畫面上檢視也能正常瀏覽使用。

### Nice to Have (可以擁有，後續擴充)
- 多使用者註冊與登入功能，讓不同使用者有獨立的代辦清單。
- 任務的優先層級設定（急迫、普通、不急）。
- 任務的到期日期（Due Date）提醒。
